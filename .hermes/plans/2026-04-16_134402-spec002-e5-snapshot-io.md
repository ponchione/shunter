## Approach
1. Read the existing snapshot implementation and tests end-to-end.
2. Compare code directly against the E5 story acceptance criteria.
3. Identify missing behaviors or stale docs.
4. Use TDD for any actual missing behavior:

   - run it to confirm RED
   - implement minimal fix
   - rerun targeted and package tests
5. Update docs (`REMAINING.md` and possibly TECH-DEBT) if the implementation is already present and the remaining-work ledger is stale.
