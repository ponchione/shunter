# Working Docs

This directory holds implementation planning, baseline specs, audits, design
notes, benchmark baselines, and roadmap material that should not sit beside the
app-author documentation in `docs/`.

## Contents

- `dependency-considerations.md` - dependency policy and dependency decisions.
- `shunter-design-decisions.md` - consolidated implementation-facing design
  decisions.
- `specs/` - numbered subsystem contracts and scope notes.
- `v1-roadmap/README.md` - active v1 implementation roadmap, auth coverage,
  hardening gate, and performance status.

Use live code and tests before working-doc prose when they disagree.
